# Contributing

感谢你愿意改进 CodexWorkspace Methodology。

## 可以贡献什么

- 新的工作区模板。
- 更清晰的 SOP。
- 项目台账字段建议。
- skills 清单维护方式。
- 问题复盘模板。
- 真实使用案例。
- 文档错别字、结构和表达优化。

## 贡献原则

- 保持 Markdown 文件简单、可复制、可手动维护。
- 不把所有规则塞进一个文件，遵守单一职责。
- 模板和文档中不要包含真实密钥、账号、私有路径或内部项目名。
- 作用说明尽量写成中文短句，避免直接复制工具 frontmatter。
- 新规则必须说明触发条件：什么时候需要更新、由谁更新、更新到哪里。

## 提交前检查

- README 链接可用。
- 模板没有私人信息。
- 路径和日期使用 `<WORKSPACE_ROOT>`、`YYYY-MM-DD` 等可替换占位符。
- 新增文件有清楚用途。
- 新增文件有清楚用途，且没有制造重复信息源。

## Issue 建议格式

```markdown
## 背景

说明你遇到的工作区管理问题。

## 建议

说明你希望增加、删除或调整什么。

## 预期效果

说明这个改动如何减少混乱、返工或维护成本。
```
