# Codex 执行规则

`AGENTS.md` 是 Codex 的强约束文件。它应该短、硬、明确。

## 应该包含

- 工作区边界：所有工作只在固定工作区中完成。
- 外部项目规则：先说明路径和迁移目的，获得同意后复制到工作区。
- 项目命名规则：引用命名细则文件，不重复展开。
- 工作流程：修改前先理解结构，不改无关文件。
- 文档维护：出错后更新日志，项目变更后更新台账。
- Codex skills 全局配置变更：安装、删除、重命名、调整或个性化修改 skill 前说明来源、路径和影响，完成后更新 skills 清单。
- 自动化与外部投递：定时任务、外部通知、报告发送前要验证真实链路、确认信息披露边界，并检查客户端展示效果。
- 安全规则：不做破坏性操作，不回退用户已有改动。

## 不应该包含

- 完整目录架构说明。
- 长篇项目 SOP。
- skills 清单正文。
- 具体某个 skill 的安装记录和完整来源列表。
- 项目台账。
- 复盘日志正文。
- 具体外部通道的完整调试过程。

这些内容应该放到对应文件中，`AGENTS.md` 只负责让 Codex 知道必须遵守什么。

## 全局规则与工作区规则

可以在 Codex 全局规则中只放一个轻量边界：

```text
所有 Codex 项目工作必须先迁移到固定工作区。
```

进入工作区后，再读取工作区内的 `AGENTS.md`。

项目中的 `templates/global-codex-custom-instructions.template.md` 提供了一个更完整的全局自定义指令模板，可复制到 Codex 的「设置 -> 个性化 -> 自定义指令」。该模板只负责全局边界和治理触发器，具体项目规则仍由工作区内的 `AGENTS.md` 承载。
