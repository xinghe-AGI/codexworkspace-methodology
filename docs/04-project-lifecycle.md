# 项目生命周期

一个正式项目在 CodexWorkspace 中通常经历五个阶段。

## 1. 新建或迁移

新项目直接在 `projects/` 下创建。外部项目先获得用户同意，再复制到 `projects/` 下。

项目目录命名遵守：

```text
Project_三位编号_项目名称
```

## 2. 初始化

初始化阶段至少要明确：

- 项目类型
- 项目用途
- 目录结构
- 使用方式
- README 是否需要补充

代码项目可能需要安装、运行、测试说明；知识库项目可能更需要主题范围、分类方式和检索方式。

## 3. 使用与维护

维护时优先遵守项目已有结构，不做无关重构。项目入口、使用方式或状态变化时，更新 `projects-index.md`。

## 4. 问题复盘

如果 Codex 出现错误、不满意、返工或踩坑，并且问题已经处理完成，应更新 `codex-issue-log.md`。

## 5. 归档

项目停止推进时移动到 `archives/`，项目编号不复用，`projects-index.md` 中状态改为 `归档`。
