# GitHub 发布清单

发布前检查以下事项。

## 内容检查

- README 能解释项目是什么、适合谁、怎么开始。
- docs 中没有只适用于个人电脑的绝对路径，或已说明需要替换。
- templates 中不包含隐私信息、账号、密钥、内部项目名。
- 示例路径和命名规则清晰。

## 隐私检查

- 搜索并移除真实密钥、账号、token。
- 搜索并移除不应公开的本地路径。
- 确认复盘日志中没有私人项目细节。

## 许可证

当前项目使用 MIT License，发布前需要确认 `LICENSE` 文件仍然存在，且 README 中的 License 说明与实际许可证一致。

## 仓库设置

- 仓库名建议：`codexworkspace-methodology`
- 描述建议：`A workspace methodology for managing Codex projects, rules, SOPs, skills, and issue logs.`
- Topic 建议：`codex`、`ai-agents`、`workflow`、`documentation`、`project-management`
