# 自动化台账

最后更新时间：YYYY-MM-DD

> 本文件同时给人和 Codex 读取：人先看“总览”，快速判断有哪些自动化、是否启用、产出给谁；Codex 再按自动化编号读取对应“场景详情”，执行、排错或维护。
> 不要在本文件中记录 App Secret、token、邮箱密码、GitHub 个人访问令牌或其他完整凭据。

## 维护规则

- 创建、更新、暂停、恢复或删除自动化后，同步更新本台账。
- 每个自动化必须有唯一编号，例如 `A001`、`A002`。
- 总览表只放人需要快速扫描的信息；不要把所有字段塞进总览表。
- 每个自动化都必须有独立的“场景详情”，写清目标、触发、输入、输出、权限、失败处理和 Codex 执行步骤。
- 自动化涉及外部投递时，记录接收方类型和占位标识即可，例如 `飞书群：<FEISHU_CHAT_NAME_OR_ID>`。
- 自动化读取本地文件并外发前，必须先确认信息披露边界。
- 长报告默认使用“短摘要 + 附件”，不要把完整正文塞进单条聊天消息。
- 自动化出现真实投递失败、客户端展示异常、披露风险拦截或权限问题时，处理完成后记录到 `codex-issue-log.md`。

## 状态说明

- `启用`：自动化按计划运行。
- `暂停`：自动化保留但暂不运行。
- `停用`：自动化不再使用，仅保留历史记录。

## 场景详情

### A001 Codex 错误复盘周报

#### 人读摘要

- 目的：定期整理 `codex-issue-log.md` 中的相似错误、重复复盘、复发模式和防复发待办，避免同一问题被反复发送给用户。
- 读者：工作区维护者、需要了解 Codex 近期失误模式的人。
- 成功标准：用户收到一条可读短摘要，并能打开完整报告附件；报告能指出哪些复盘条目疑似重复、建议合并到哪一条、需要保留哪些发生证据，以及下一步改进建议。
- 不做什么：不在每轮最终回复前强制检查旧问题；不自动修改 `AGENTS.md`、`codexworkspace-console.md`、`codex-issue-log.md` 或其他治理文件，除非用户明确授权合并或改写。

#### 触发与时间

- 类型：定时复盘。
- 频率：每周一次。
- 时间：每周一 21:00。
- 时区：Asia/Shanghai。
- 暂停条件：飞书通道不可用、用户暂停外部投递、报告涉及未获批准的敏感信息。

#### 读取范围

- 工作目录：`<WORKSPACE_ROOT>`。
- 必读文件：
  - `codex-issue-log.md`
  - `AGENTS.md`
- 可选参考：
  - `codexworkspace-console.md`
  - `codex-skills-inventory.md`
- 排除内容：密钥、token、App Secret、个人访问令牌、完整私有凭据。

#### 输出与投递

- 本地完整报告：`temp\codex-error-audit-report-YYYY-MM-DD.md`。
- 聊天流摘要：单行短摘要，只写本周核心发现和附件提示。
- 外部投递：飞书群 `飞书群：<FEISHU_CHAT_NAME_OR_ID>`。
- 投递格式：短摘要 + `.md` 或 `.txt` 附件。
- 展示验证：不能只看 API 返回成功；需要确认客户端能看到摘要并能打开附件。
- 保留策略：`temp\` 中报告可按工作区实际习惯清理或归档。

#### 权限与安全

- 允许读取：本场景“读取范围”中列出的治理文件。
- 允许写入：仅写入 `temp\codex-error-audit-report-YYYY-MM-DD.md`。
- 不允许写入：默认不修改治理文件、项目文件或全局 Codex 配置。
- 外部披露：发送前必须确认报告内容可以发给目标接收方。
- 凭据规则：凭据只来自本机安全配置或环境变量，不写入本台账和报告。

#### Codex 执行步骤

1. 读取 `codex-issue-log.md` 和 `AGENTS.md`。
2. 识别疑似重复复盘：标题相近、触发场景相同、原因相同、处理规则相同或后续规则高度重合的条目。
3. 对每组重复问题输出建议主条目、建议合并条目、应保留的发生记录和建议状态标注。
4. 按“重复复盘分组、复发模式、已有规则覆盖、缺失防线、下周待办”整理报告。
5. 将完整报告写入 `temp\codex-error-audit-report-YYYY-MM-DD.md`。
6. 生成一条单行摘要，避免多行正文直接进入聊天消息。
7. 如果需要外部投递，先确认披露边界，再发送摘要和附件。
8. 发送后检查客户端展示效果；失败时说明失败原因和下一步处理建议。

#### 失败处理

- 读取文件失败：报告失败原因、涉及路径和是否需要用户授权。
- 外部发送失败：区分代理、沙箱网络、凭据、权限、目标群或客户端展示问题。
- 附件不可读：改用 `.txt` 格式补发，或在 Codex 中返回本地报告路径。
- 披露边界不明确：停止外发，等待用户明确批准。

#### 维护信息

- 关联文档：`docs/06-automation-and-external-delivery.md`、`codex-issue-log.md`。
- 最后验证：YYYY-MM-DD。
- 下次复核：YYYY-MM-DD。
- 备注：示例场景，实际使用时请替换群名、时间、验证日期和本地路径策略。
