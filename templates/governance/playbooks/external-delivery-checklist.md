# External Delivery Checklist

最后更新时间：YYYY-MM-DD

本清单适用于飞书、邮件、GitHub、Slack、网页和其他外部系统发送。

## 发送前必须确认

- 披露批准：用户是否已经明确批准把哪类本地内容发送到哪个外部目标。
- 接收目标：是否已有明确 `chat_id`、`open_id`、邮箱地址、仓库、链接或其他精确目标；不要靠枚举、猜测或模糊描述推断。
- 发送身份：是否明确用 `bot` 还是 `user` 身份，权限和可见范围是否匹配。
- 网络链路：是否需要禁用代理、走直连或使用指定命令入口；例如飞书 Bot 固定检查 `LARK_CLI_NO_PROXY=1` 和 `.cmd/.exe`。
- 消息形态：客户端是否适合当前内容；长报告不要默认塞进单条 Markdown 或多行 text。

## 飞书默认做法

- 长报告默认使用“单行摘要 + 附件”。
- 不使用 `--markdown` 承载完整正文。
- 不把完整报告塞进多行 `--text`。
- 发送后以客户端展示效果为准，不只看 `ok:true`。

## 发送最小链路

1. 明确本地内容边界。
2. 明确接收目标。
3. 明确发送身份。
4. 用最小内容做真实发送或真实调用验证。
5. 确认客户端可读，再发送正式内容。

## 需要停下来的情况

- 用户只批准了通道配置，没有批准具体内容外发。
- 目标对象不明确，只有“发给我”“发群里”这类描述。
- 只能看到 API 成功，无法证明客户端可读。
- 当前身份可见范围不足，必须靠枚举无关资源才能定位目标。
